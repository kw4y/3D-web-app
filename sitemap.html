Site Map

The site is structured as a single-page desktop environment. The home route (index.html) loads the desktop with the Three.js wallpaper scene; each app in the dock opens its own window inside that desktop. The three standalone HTML pages (submission, sitemap, references) are real files at the site root and can be accessed either directly via URL or through the Safari tabs.

Routes

  • /index.html - Desktop home (Three.js wallpaper + dock)
    • Wallpaper scene - grass field, character, post-processing
    • Menu bar - Finder-style top bar with clock
    • Desktop widgets - Clock, Brighton weather (Open-Meteo), calendar
    • Dock - launches apps below
      • Models - Blender-style 3D model viewer (Character / Apple / Headphones / CD)
      • About - Notes-style writeup of the project
      • Safari - this browser shell, embeds the pages below
      • Spotify - mini-player embedding the Lily Chou-Chou album (audio integration)
      • Bin - decorative, not interactive
  • /submission.html - Required submission evidence (URLs, statements, testing)
  • /sitemap.html - This page
  • /references.html - Attributions and credits

Code Architecture

  • /js
    • main.js - entry point, initialises scene, dock, clock and desktop widgets
    • /three - 3D scene code
      • scene.js- wallpaper scene
      • modelviewer.js- per-window 3D viewer
    • /ui - DOM controllers
      • window.js- window factory
      • dock.js- dock controller, app registry
      • widgets.js- desktop Clock / Weather / Calendar widgets
      • /apps
        • models.js
        • about.js
        • safari.js
        • spotify.js
  • /css - stylesheets, modular per app
    • desktop.css- wallpaper, menu bar, dock, loader
    • window.css- window chrome + per-app stylesheets
    • widgets.css- frosted-glass desktop widgets
    • canvas.css- shared Canvas LMS theme for the three standalone pages
  • /assets - icons, 3D models (.glb, incl. the textured CD), source .blend files, university logo
