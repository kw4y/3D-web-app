Deeper Understanding

This project builds on the lab tutorials in several ways. Instead of creating a brand-focused product showcase, I designed a desktop environment inspired by macOS. In this setup, the 3D models are part of an interactive operating system metaphor. Each app in the dock opens its own draggable window.

I chose Three.js instead of X3DOM to get native WebGL access. The home screen features a Three.js scene with a grass field, a custom character whose head follows the mouse using bone manipulation, and a bloom effect for a dreamlike look. The Models app works like a Blender viewer, offering four shading modes (Wireframe, Solid, Material Preview, and Rendered), a custom view-helper gizmo, and switchable Blender and Showcase camera modes. Each mode shows different lighting setups and camera types, as required by the rubric.

I created all assets in Blender. The .blend files are stored in /assets/Models/ in the repository. I textured them with PBR materials and exported them as glTF Binary (.glb) files with embedded textures. All file paths are case-sensitive to work properly on the ITS Linux server.

Testing feedback was limited to checking the project on three different browser and device combinations. The main gap in my submission is the lack of external user testing, which I discuss in more detail in the User Testing section below.

User Testing

To be honest, I ran out of time and couldn't get anyone outside the cohort to test the project. The rubric asks for user testing with three people who are not part of the class, observing them as they use the project and recording their feedback and impact statements. I was not able to do this. The only testing I managed was checking the project on the devices I already had.

For my testing strategy, I loaded the live build on three browser and device combinations to find any cross-platform issues. I used Chrome and Firefox on Windows, and Safari on macOS. I tested each app in the dock by opening and closing windows, dragging them around, switching models in the Models app between the four shading modes, and clicking through the Safari tabs to make sure iframe swapping worked.

A key weakness is that cross-browser checks are not the same as real user testing. Without outside testers, I do not know if first-time visitors understand that the desktop metaphor is interactive, if the dock icons make sense without labels, or if anything I built is confusing in ways I cannot see as the creator. The rubric asks for feedback and impact statements, but I do not have these. This is the biggest gap in my submission.
